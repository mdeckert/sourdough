Key Features
============

Server (HTTP API)
-----------------
- Endpoints: /health, /bake/start, /log/{event}, /log/temp/{value}, /status
- Runs on port 8080 (configurable)
- Lightweight, suitable for 24/7 operation
- Stores data in JSON Lines format

CLI Tool
--------
- Commands: start, log, temp, status, complete, history, review
- Interactive assessments
- Formatted timeline output with durations
- Works via SSH

QR Code System
--------------
- Generate codes for all common events
- Point to HTTP endpoints
- Scan with phone → instant logging
- No app required!

Data Storage
------------
- One .jsonl file per bake
- Human-readable JSON format
- Easy to backup and analyze
- Example: bake_2025-10-07.jsonl

Workflow
========

1. Install: ./install.sh
2. Start bake: sourdough start (or scan QR)
3. Log events: sourdough log fold (or scan QR)
4. Check status: sourdough status
5. Complete: sourdough complete (assessment)
6. Review: sourdough history / sourdough review 2025-10-07

Event Timeline
==============

starter-out → fed → levain-ready → mixed → fold (x3-4) → shaped 
   → fridge-in → [overnight] → fridge-out → oven-in → bake-complete

Temperature tracking throughout for timing correlation analysis.

Future Enhancements
